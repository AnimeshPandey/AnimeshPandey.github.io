How Well Do You Know this?

February 28, 2021 9 min read JavaScript this keyword Binding Rules

The this keyword in JavaScript is one of the language's most misunderstood and most frequently Googled concepts. Developers often treat it as magic or memorise fragile heuristics. This article builds a correct mental model from first principles.

The most important thing to understand: this is not author-time binding — it's runtime binding. It has nothing to do with where a function is defined and everything to do with how and where it is called. The call-site — the location in code where a function is invoked — determines what this refers to.

JavaScript has exactly four binding rules for this, applied in a strict order of precedence. Understanding the rules — and their precedence — lets you look at any call-site and know with certainty what this will be, without guessing. This is the mental model the article builds.

Rule 1 — Highest precedence
new Binding
When a function is called with new, this is the newly created object.
Rule 2
Explicit Binding
call(), apply(), or bind() explicitly set this to the passed object.
Rule 3
Implicit Binding
When a function is called as a method on an object, this is that object.
Rule 4 — Lowest precedence
Default Binding
When no other rule applies, this is the global object (or undefined in strict mode).
